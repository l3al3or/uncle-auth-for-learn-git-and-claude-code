# Feature Request: ล็อกบัญชีชั่วคราวเมื่อ login ผิดหลายครั้ง

## ที่มา (Why)
ตอนนี้ใครก็ตามสามารถเดารหัสผ่านของผู้ใช้ได้ไม่จำกัดจำนวนครั้ง
(brute-force attack) ต้องเพิ่มการป้องกันขั้นพื้นฐาน

## สิ่งที่ต้องการ (What)
- ถ้า login ผิดติดต่อกันครบ 5 ครั้ง ให้ "ล็อก" บัญชีนั้นชั่วคราว 15 นาที
- ระหว่างถูกล็อก แม้กรอกรหัสถูก ก็ยังเข้าไม่ได้ และต้องแจ้งผู้ใช้ว่าบัญชีถูกล็อก
- ถ้า login สำเร็จ ให้รีเซ็ตตัวนับกลับเป็น 0

## เกณฑ์ผ่าน (Acceptance Criteria)
1. login ผิด 5 ครั้ง -> ครั้งที่ 6 ถูกปฏิเสธแม้รหัสถูก พร้อมข้อความว่าบัญชีถูกล็อก
2. login ผิด 4 ครั้งแล้วครั้งที่ 5 ใส่ถูก -> เข้าได้ และตัวนับกลับเป็น 0
3. test เดิมทั้ง 10 ตัวต้องไม่แดง

## หมายเหตุสำหรับการทำ (แนวทาง)
- เก็บ failed_attempts และ locked_until ไว้กับ user แต่ละคนใน users.json
- เขียน test ก่อน (TDD) ให้แดงก่อน แล้วค่อย implement ให้เขียว
